## `genesis`[](https://docs.oasis.io/core/oasis-node/cli#genesis)

### `check`[](https://docs.oasis.io/core/oasis-node/cli#check)

要检查给定的创世文件是否有效，请运行：

```
oasis-node genesis check --genesis.file /path/to/genesis.json

```

INFO

这还会检查 genesis 文件是否为规范形式。

### `dump`[](https://docs.oasis.io/core/oasis-node/cli#dump)

在特定的区块高度转储网络的状态，例如 717600，到创世文件，运行：

```
oasis-node genesis dump \
  --address unix:/path/to/node/internal.sock \
  --genesis.file /path/to/genesis_dump.json \
  --height 717600

```

CAUTION

您只能在网络上达到给定的块高度后运行以下命令。

### `init`[](https://docs.oasis.io/core/oasis-node/cli#init)

要使用给定的链 id 和质押代币符号初始化新的创世文件，请运行：

```
oasis-node genesis init --genesis.file /path/to/genesis.json \
  --chain.id "name-of-my-network" \
  --staking.token_symbol TEST

```

INFO

您可以为各种共识层服务设置很多参数。

要查看完整列表，请运行：

```
oasis-node genesis init --help

```

## `stake`[](https://docs.oasis.io/core/oasis-node/cli#stake)

### `account`[](https://docs.oasis.io/core/oasis-node/cli#account)

### `info`[](https://docs.oasis.io/core/oasis-node/cli#info)

运行

```
oasis-node stake account info \
  --stake.account.address <account address> \
  --address unix:/path/to/node/internal.sock

```

### `pubkey2address`[](https://docs.oasis.io/core/oasis-node/cli#pubkey2address)

运行

```
oasis-node stake pubkey2address --public_key <public_key>

```
